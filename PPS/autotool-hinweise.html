Aufgabe 42-3 (Formel -> Regexp)

siehe Reguläre Ausdrücke 👆

Vorlesung: Skript

Beispiel:

forall p : ((a (p) => exists q : (((p < q) && b (q)))))

→ Für alle Stellen p gilt: Wenn an der Stelle ein a steht dann existiert eine Stelle q, für die gilt: die Stelle p ist vor (kleiner) der Stelle q und an der Stelle q steht ein b.

Lösung:

((a^*c^*)^*bc^*)^*

Prädikatenlogik

  • Implikation
    • Implikation → x => y → wenn x, dann y
    • Wenn x falsch ist, ist Implikation immer wahr
    • nur wenn x wahr ist, dann hängt Implikation von y ab
  • Konjuktion
    • Und → x ∧ y
  • Disjunktion
    • Oder → x ∨ y

Hinweise:

  • es gibt Buchstaben innerhalb des Alphabetes → können in Wort vorkommen
  • es gibt Buchstaben nicht im Alphabet → Positionen im Wort
  • a § meint dabei eine Position p im Wort mit dem Buchstaben a
  • b (q) meint dabei eine Position q im Wort mit dem Buchstaben b
  • p < q → der Buchstabe a kommt vor dem Buchstaben b

Aufgabe 53-1 (Frames)

Vorlesung: Skript
Lösung:

{
    a = function() {b();};
    b = function() {
        c = function() {
            halt;
        };
        c();
    };
    a();
}

Vorgehensweise:

  1. Frames/Graphen zeichnen
  2. Aufgrund von statischen Vorgänger die textuellen Deklarationen der Funktion identifizieren
  3. Aufgrund des dynamischen Vorgängers Funktionen entsprechend aufrufen
    1. Aufrufe auch per Parameter möglich
    2. Aufrufe auch durch Rückgabe einer Funktion möglich

Hinweis:

  • dynamischer Vorgänger → Frame des aufrufenden Unterprogramms/Funktion
  • statischer Vorgänger → Frame des textuell umgebenen Unterprogramms in Code
  • in Aufgabe ist erster Frame der main-Frame → die umrandende Klammer ( { )
  • erster Frame hat Vorgänger zum OS
  • Verwendung von halt → Beenden des Programmes
  • Semikolons nicht vergessen: nach Aufruf, Deklaration
  • Rückgabe von Funktionen durch Hinschreiben des names f = function() {g;};
  • Funktionen können Parameter haben
  • zurückgegebene Funktion kann beispielsweise so aufgrufen werden: f()()

Aufgabe 54-1 (Überladung auflösen)

Vorlesung: Skript, weiterführende Literatur
Lösung:

Solution
    { program = Program
                    { classes = [ class F extends C
                                , class C
                                , class D extends E
                                , class E extends C
                                ]
                    , methods = [ m ( E x, D y, C z)
                                , m ( F x, D y, D z)
                                , m ( D x, F y, F z)
                                , m ( E x, E y, E z)
                                , m ( C x, C y, E z)
                                ]
                    , call = m ( new E () , new D () , new D () ) 
                    }
    , minimal = [ 3, 0] 
    }

Vorgehensweise:

  1. Code kopieren und formatieren
  2. Hierarchiebaum zeichnen → meistens Verzweigung oder Reihenfolge
  3. So viele Funktionen wie möglich nicht passend (not applicable) gestalten
  4. Funktionendeklarationen und Aufrufe mit Blatt testen → minimale Funktionen ermitteln (siehe Hinweis)
  5. Anpassungen treffen und minimale Funktionen neu bestimmen

Hinweis:

  • Zahl im Minimal-Array ist Index der Funktion im Array methods
  • Minimale Methode ermitteln
    • P ist Menge der möglichen (applicable) Methoden
    • M ist die Menge der minimalen Methoden
    • Funktion ist minimal, wenn ‘¬∃p∈P:p<m‘`\lnot \exist p \in P: p < m`‘¬∃p∈P:p<m‘
    • d.h. es existiert in P keine Funktion die kleiner ist als die zu prüfende Funktion
    • kleiner bedeutet hier, dass jeder Typ kleiner oder gleich (weiter unten im Hierarchiebaum) als der Typ der zu prüfende Methode ist.

Beispiel für Minimal:

  • Hierarchiebaum: C → D → E → F
  • Menge P: [ ( D,F,E ), (C, E, C) ]
    • Methode (D,F,E) minimal? → (C,E,C) < (D,F,E)? → C > D, E > F, C > E → (C,E,C) nicht kleiner → minimal
    • Methode (C,E,C) minimal? → (D, F, E) < (C, E, C)? → D < C, F < E, E < C → (D,F,E) kleiner → nicht minimal
  • Menge P: [ (D, E, F), (C, F, D) ]
    • Methode (D, E, F) minimal? → (C, F, D) < (D, E, F)? → C > D → (C, F, D) nicht kleiner → minimal
    • Methode (C, F, D) minimal? → (D, E, F) < (C, F, D) ? → D < C, E > F → (D, E, F) nicht kleiner → minimal

Eigener/Schnellerer Lösungsweg:

  • Distanz berechnen via Hierarchiebaum
  • Für jeden Schritt zum Eltern-Knoten -1 berechnen
  • Aufaddieren der Gesamtschritte für einen Parameter bis Zielknoten erreicht
  • Aufaddieren der Gesamtschritte aller Parameter
  • Funktion minimaler, wenn näher an Null
  • Überprüfung mit der richtigen Methode oder Compiler (siehe Autotool-Tests)
  • Beispiel
    • Hierarchiebaum: C → D → E → F
    • Methodendeklaration: m (D x, F y, C z)
    • Aufruf m( new F(), new F(), new F() )
    • Distanz für ersten Parameter → -1 + -1 → -2
    • Distanz für zweiten Parameter → 0 → 0
    • Distanz für dritten Parameter → -1 + -1 + -1 → -3
    • Gesamtdistanz → -2 + 0 + -3 → -5

Vorlesung: Skript

Lösung:

S.<Animal>rowlf(S.<Animal>statler(S.<Piggy<Animal>>statler(S.<Animal>bunsen(S.waldorf(), S.chef()))), S.chef())

S.<Animal>rowlf(
	S.<Animal>statler(
		S.<Piggy<Animal>>statler(
			S.<Animal>bunsen(
				S.waldorf(),
				S.chef()
			)
		)
	),
	S.chef()
)

Vorgehensweise:

  1. Code kopieren und formatieren
  2. Identifizieren der Methoden ohne Parameter → innerste Methoden
  3. Identifizieren der Methoden mit richtigem Rückgabetyp → äußerste Funktion
  4. Anfang von innen nach außen
    1. Welcher Typ gibt die innerste Methode (Schritt 2) zurück
    2. Wo kann dieser Rückgabetyp eingesetzt werden
    3. Welche weiteren Parameter muss ich berücksichtigen

Hinweis: Wichtig sind die expliziten Typangaben bei Funktionen, die einen generischen Typ erwarten. Syntax: ClassName.<Type>methodName
